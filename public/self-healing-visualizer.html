Self-Healing Locator Framework

When a Playwright locator matches zero elements, the framework snapshots the interactive DOM, asks xAI (Grok) for a repaired selector under a strict JSON schema, re-validates that selector against the live page, and only then continues — attaching the whole trace to the test report. This page animates that flow for the naive v1 helper and the centralized v2 service.

Framework event log

Shipped in v3

  • Confidence gate: a repair is scored from uniqueness (matches exactly one element) and selector stability (id/data-testid > role/text > nth-child), capped by the model's own number. Below 0.5 the service refuses to heal instead of clicking a guess. src/framework/self-healing/confidence.ts
  • Selector cache: a SQLite cache keyed on (page_key, broken_selector, intent) reuses a previously healed selector so xAI is called once, not every run — still re-validated against the live DOM before use. src/framework/self-healing/LocatorCache.ts

Still planned

  • Refusal rules: never heal destructive actions (delete, pay, submit-order) without an explicit allow-list.
  • Report trends: attach heal timelines to the HTML report and chart selector drift across runs.

Architecture

One pipeline, deterministic-first

Every input flows left to right. The LLM sits at the far end — reached only when cheaper, deterministic stages can't resolve it — and its output is always verified before use.

01 · TRIGGER

Break or failure

A locator matches 0 elements, or a test fails on its final retry.

02 · HOOKS

Capture (MITM)

Fixtures + a recording API context intercept DOM, screenshot, and the HTTP exchange.

03 · DETERMINISTIC

Resolve locally

Cache, local locator rebuild, and regex PII redaction — no model, no tokens.

04 · LLM (JUDGE)

Escalate to Grok

Only for the genuinely hard cases: repair, classify, or a local judge for PII.

05 · GUARDRAILS

Verify · attach

Confidence + contract gate the answer; refuse or heal; explanation on the report.

💸 Token economics — cache, deterministic-first, dedup, budgets. The model is the expensive last resort.
🔒 Data security — PII redacted (patterns + local judge) before anything remote; secrets dropped at capture.
🔭 Observability — every LLM call logged with latency, tokens, and cost to xai-calls.jsonl.
What makes it different

Six ideas doing the heavy lifting

🧠

LLM as a Judge

The model ranks and rules; it isn't trusted blind. Confidence caps the model's self-report with measured signals, failures are classified into four buckets, and a local model judges residual PII.

confidence · classification · PII judge
🪝

Hooks & MITM

Auto-fixtures wrap the test lifecycle and a recording wrapper sits in front of APIRequestContext — a man-in-the-middle that captures every request/response without changing a single test.

fixtures · RecordingApi · DOM capture
💸

Token economics

Deterministic resolution, a SQLite locator cache with multi-locator fallback, and post-run cascade dedup mean most breaks cost zero tokens and a 40-test cascade collapses to one call.

cache · deterministic-first · dedup
🔒

User-data security

Nothing reaches the remote model unscrubbed. Pattern-based redaction (field-aware for secrets) runs over the full text; the PII judge runs locally so data isn't shipped off-box just to be detected.

redaction · field-aware · local judge
🔭

Observability

Every real LLM call is routed through one method that logs latency, HTTP status, token usage, and cost — a full audit trail for a non-deterministic dependency.

xai-calls.jsonl · XAI_DEBUG
🛡️

Guardrails & refusal

A confident-but-wrong heal is refused, not clicked. A target contract (role/type/ancestor) plus a confidence threshold turn "trust the model" into "verify the model".

contract · threshold · refuse-on-ambiguity

Evolution

Eleven tested steps

Each iteration solves the honest limitation of the one before it.

v1

Naive healing

Broken locator → capture DOM → xAI repair → validate → continue.

v2

Centralized service + fixture

One SelfHealingService, shared via a Playwright fixture — tests stop wiring helpers.

v3

Confidence gate + SQLite cache

Refuse low-confidence heals; call the model once per break, reuse from cache.

v4

Contract disambiguation

Verify the model's answer against a role/type/ancestor contract — refuse the confident-but-wrong element.

v5

Deterministic-first

Rebuild the locator locally; escalate to the LLM only when genuinely ambiguous. Most breaks: 0 tokens.

v6

Multi-locator cache

Store the target's top-3 attribute-diverse locators; single-attribute drift heals from a fallback.

v7

Failure Explainer (Task 3 Option A)

On failure → page state + error → categorized explanation attached to the report.

v8

Vision

Attach a page screenshot — the model sees the render, not just the DOM text.

v9

Batched + enriched (mode B)

Post-run reporter dedups cascades and enriches with run history + git diff to ground the classification.

v10

API request/response

For API failures, send the raw HTTP exchange — there is no DOM to send.

v11

PII redaction

Scrub context before the remote LLM: patterns + field-aware secrets + a local LLM judge.

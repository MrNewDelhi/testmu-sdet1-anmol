Self-Healing Locator Framework

When a Playwright locator matches zero elements, the framework snapshots the interactive DOM, asks xAI (Grok) for a repaired selector under a strict JSON schema, re-validates that selector against the live page, and only then continues — attaching the whole trace to the test report. This page animates that flow for the naive v1 helper and the centralized v2 service, and shows how v4 refuses a confident-but-wrong heal that violates the target contract.

Shipped (v1–v7)

  • v1–v2: naive xAI locator repair, centralized into a shared service + fixture.
  • v3: programmatic confidence gate + SQLite locator cache.
  • v4: target-contract disambiguation (refuse the confident-but-wrong element).
  • v5: deterministic-first resolution to avoid the LLM on easy cases.
  • v6: multi-locator cache so single-attribute drift heals from a fallback.
  • v7: Failure Explainer (Task 3 Option A) — on failure, send page state + error to xAI and attach a categorized explanation to the report.

Still planned

  • v7 mode B: move the LLM call to a post-run reporter — batch all final failures, dedup by error signature, one grouped report.
  • Refusal rules: never heal destructive actions (delete, pay, submit-order) without an explicit allow-list.
  • Report trends: attach heal timelines to the HTML report and chart selector drift across runs.
